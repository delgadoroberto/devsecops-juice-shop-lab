# Contributing

Thank you for your interest in contributing to this project.

This repository is an educational DevSecOps laboratory designed to demonstrate automated security testing using GitHub Actions, Trivy, and OWASP Juice Shop.

Contributions that improve the educational value, documentation, automation, or overall quality of the project are welcome.

---

## Ways to Contribute

You can contribute by:

- Improving documentation
- Fixing errors or typos
- Enhancing GitHub Actions workflows
- Improving Docker configuration
- Adding new security automation examples
- Expanding remediation guidance
- Suggesting new DevSecOps best practices

---

## Development Guidelines

Please follow these guidelines before submitting changes:

- Keep the repository focused on its educational purpose.
- Follow existing Markdown formatting.
- Write clear and concise commit messages.
- Test changes before submitting them.
- Ensure GitHub Actions complete successfully.

---

## Pull Request Process

1. Fork the repository.
2. Create a feature branch.

```text
feature/your-feature-name
```

3. Make your changes.
4. Verify that all GitHub Actions workflows pass.
5. Submit a Pull Request with a clear description of your changes.

---

## Coding Standards

When contributing:

- Keep examples simple and easy to understand.
- Prefer secure coding practices.
- Avoid unnecessary complexity.
- Maintain consistent formatting throughout the repository.

---

## Documentation Standards

Documentation should:

- Use clear technical English.
- Include practical examples where appropriate.
- Remain consistent with the project's educational objectives.
- Be easy to follow for beginners and intermediate learners.

---

## Reporting Issues

If you discover a bug or documentation issue, please open a GitHub Issue including:

- Description
- Expected behavior
- Actual behavior
- Steps to reproduce
- Suggested improvement (optional)

---

By participating in this project, you agree to follow the project's Code of Conduct.
