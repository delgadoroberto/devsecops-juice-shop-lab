A practical DevSecOps laboratory that demonstrates how automated security controls can be integrated into a Continuous Integration (CI) pipeline using GitHub Actions.

This project uses **OWASP Juice Shop**, a deliberately vulnerable web application, together with **Trivy** to perform automated security scans against container images, the project filesystem, and hardcoded secrets. The goal is to illustrate how security testing can be shifted left and incorporated into modern software development workflows.

# Overview

Modern software development requires security validation to occur throughout the software lifecycle instead of only during production deployments.

This laboratory demonstrates a simple but effective DevSecOps pipeline capable of automatically identifying:

- Vulnerable container images
- High and Critical CVEs
- Hardcoded secrets
- Insecure project files

The repository is intentionally small and focused on demonstrating the integration of automated security testing within GitHub Actions.

---

# Objectives

The primary objectives of this laboratory are:

- Understand Shift-Left Security principles.
- Integrate security testing into CI pipelines.
- Perform automated container image scanning.
- Detect hardcoded credentials.
- Analyze vulnerability reports generated by Trivy.
- Learn basic remediation strategies.

# Security Controls

The pipeline currently performs the following automated controls.

## Container Image Scan

Scans the Juice Shop Docker image for known vulnerabilities.

Focus:

- HIGH vulnerabilities
- CRITICAL vulnerabilities

---

## Filesystem Scan

Scans the repository contents for:

- vulnerable dependencies
- configuration issues
- local files

---

## Secret Detection

Searches the repository for accidentally committed credentials such as:

- AWS Keys
- GitHub Tokens
- API Keys
- Passwords
- Private Keys

The repository intentionally includes a sample secret for educational purposes.

---

# CI/CD Pipeline

The GitHub Actions workflow performs the following stages:

1. Checkout repository
2. Pull Juice Shop image
3. Scan Docker image
4. Scan repository filesystem
5. Detect secrets
6. Generate reports
7. Fail the pipeline if High or Critical vulnerabilities are found

# Getting Started

Clone the repository.

```bash
git clone https://github.com/YOUR_GITHUB_USERNAME/devsecops-juice-shop-lab.git
```

Start the laboratory.

```bash
docker compose up -d
```

Open your browser.

```
http://localhost:3000
```

---

# Running the Lab

Push any change to the repository.

GitHub Actions will automatically execute the security pipeline.

The workflow performs:

- Image scanning
- Filesystem scanning
- Secret detection

Review the workflow logs for detailed findings.

# Remediation

Example remediation actions include:

- Remove hardcoded credentials.
- Rotate exposed secrets.
- Upgrade vulnerable container images.
- Update dependencies.
- Store sensitive values in GitHub Secrets.
- Apply least privilege principles.
